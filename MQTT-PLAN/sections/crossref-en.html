Affected Agentc back to ToC or Class ToC

IRI: http://w3id.org/mqtt-plan#AffectedAgent

A variable describing a single agent associated with mqtt-plan:ActionResult. The mqtt-plan:BrokerAction that generated the action result must have been targeted on this agent. For example, such agent can refer to the client that was disconnected by the mqtt-plan:Disconnect action.
has super-classes
Result Attribute c

Affected Agentsc back to ToC or Class ToC

IRI: http://w3id.org/mqtt-plan#AffectedAgents

A variable describing a collection of agents associated with mqtt-plan:ActionResult. The mqtt-plan:BrokerAction that generated the action result must have been targeted on all agents who are members of this collection. For example, such collection can describe the recipients to whom the broker forwarded a received message as part of mqtt-plan:Publish action.
has super-classes
Result Attribute c

Keep Alivec back to ToC or Class ToC

IRI: http://w3id.org/mqtt-plan#KeepAlive

A variable describing a maximum time interval (meassured in seconds) between two control packets sent by a client to prevent diconnection from the broker. Note: as per MQTT standard, value 0 means that this feature should be deactivetated, however, broker still may close the connection if it determines the client is inactive or non-responsive regrdless of Keep Alive value provided.

Publishc back to ToC or Class ToC

IRI: http://w3id.org/mqtt-plan#Publish

A step describing a planned activity of forwarding a message from one client (sender) to all clients registered to receive messages under the specified topic. The sender is described as the mqtt-prov:Sender of the mqtt-prov:ControlPacket which triggers this action. The topic is described by the mqtt-plan:TopicName attribute associated with the mqtt-plan:ControlPacket triggering this action. And the message of the topic can be described as mqtt-plan:Message attribute associated with the mqtt-plan:ControlPacket triggering this action. This step represents an activity performed by the broker and implies that the message was forwarded to a single or multiple agents captured in the mqtt-plan:ActionResult of this step using mqtt-plan:AffectedAgent or mqtt-plan:AffectedAgents.
has super-classes
Broker Action c

Subscribec back to ToC or Class ToC

IRI: http://w3id.org/mqtt-plan#Subscribe

A step describing a planned activity of subscribing a client from specific topics. The client is described as the mqtt-prov:Sender of the mqtt-prov:ControlPacket which triggers this action. The topics for which the client is subscribed are described by the mqtt-plan:TopicFilter attribute associated with the mqtt-plan:ControlPacket triggering this action. This step represents an activity performed by the broker on internally maintained list of topic subscribtions, it does not imply any response is sent to the client. The acknowledgment of subscribing can be modeled with mqtt-plan:SubscribeAcknowledgment broker action triggered by the mqtt-plan:ActionResult of this step.
has super-classes
Broker Action c

has super-classes
